---
title: "Customers API"
description: "Manage customer accounts and contacts"
---

Customers represent the organizations that hire your company to move freight. Each customer record stores identity, address, geocoding status, and consolidation preferences. An optional billing profile and email profile can be embedded or retrieved separately.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/customers/` | List customers |
| `GET` | `/customers/{customerID}` | Get a customer |
| `POST` | `/customers/` | Create a customer |
| `PUT` | `/customers/{customerID}/` | Replace a customer |
| `PATCH` | `/customers/{customerID}/` | Partially update a customer |
| `POST` | `/customers/bulk-update-status/` | Bulk update customer statuses |
| `GET` | `/customers/select-options/` | List customers for select inputs |
| `GET` | `/customers/select-options/{customerID}/` | Get a single customer option |

## List customers

<ParamField query="query" type="string">
  Full-text search against `code` and `name`.
</ParamField>

<ParamField query="limit" type="number" default="10">

</ParamField>

<ParamField query="offset" type="number" default="0">

</ParamField>

<ParamField query="includeState" type="boolean" default="false">

</ParamField>

<ParamField query="includeBillingProfile" type="boolean" default="false">

</ParamField>

<ParamField query="includeEmailProfile" type="boolean" default="false">

## Create a customer

### Request body

<ParamField body="code" type="string" required>
  Short unique identifier for the customer. Between 1 and 10 characters.
</ParamField>

<ParamField body="name" type="string" required>
  Full customer name. Between 1 and 255 characters.
</ParamField>

<ParamField body="status" type="string" required default="Active">
  Record status. `Active` or `Inactive`.
</ParamField>

<ParamField body="stateId" type="string" required>
  ID of the US state for the customer's address.
</ParamField>

<ParamField body="addressLine1" type="string" required>
  Primary street address. Up to 150 characters.
</ParamField>

<ParamField body="addressLine2" type="string">
  Secondary address line. Up to 150 characters.
</ParamField>

<ParamField body="city" type="string" required>
  City name. Up to 100 characters.
</ParamField>

<ParamField body="postalCode" type="string" required>
  US postal code in `12345` or `12345-6789` format.
</ParamField>

<ParamField body="externalId" type="string">
  Optional identifier from an external system.
</ParamField>

<ParamField body="allowConsolidation" type="boolean" default="true">
  Whether shipments for this customer can be consolidated.
</ParamField>

<ParamField body="exclusiveConsolidation" type="boolean" default="false">
  When `true`, consolidated loads must only contain this customer's shipments. Requires `allowConsolidation` to be `true`.
</ParamField>

<ParamField body="consolidationPriority" type="number" default="1">
  Priority used when selecting candidates for consolidation.
</ParamField>

```bash
curl -X POST "https://api.example.com/customers/" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "ACME",
    "name": "Acme Freight Co.",
    "status": "Active",
    "stateId": "state_01J9XYZ",
    "addressLine1": "100 Commerce Blvd",
    "city": "Memphis",
    "postalCode": "38101",
    "allowConsolidation": true
  }'
```

## Bulk update status

<Note>
  This endpoint is not available to API key authentication. It requires a user session token.
</Note>

Send a list of customer IDs and a target status to update them all in a single request.

```bash
curl -X POST "https://api.example.com/customers/bulk-update-status/" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "customerIds": ["cus_01J9ABCDEF", "cus_01J9GHIJKL"],
    "status": "Inactive"
  }'
```

## Enum values

### Billing cycle values

| Value | Description |
|-------|-------------|
| `Immediate` | Invoice immediately on completion |
| `Daily` | Batch and invoice daily |
| `Weekly` | Batch and invoice weekly |
| `BiWeekly` | Invoice every two weeks |
| `Monthly` | Invoice monthly |
| `Quarterly` | Invoice quarterly |
| `PerShipment` | One invoice per shipment |

### Payment term values

| Value | Description |
|-------|-------------|
| `Net10` | Payment due in 10 days |
| `Net15` | Payment due in 15 days |
| `Net30` | Payment due in 30 days |
| `Net45` | Payment due in 45 days |
| `Net60` | Payment due in 60 days |
| `Net90` | Payment due in 90 days |
| `DueOnReceipt` | Payment due on receipt of invoice |

### Credit status values

| Value | Description |
|-------|-------------|
| `Active` | Account in good standing |
| `Warning` | Approaching credit limit |
| `Hold` | Account on credit hold |
| `Suspended` | Account suspended |
| `Review` | Account under credit review |
